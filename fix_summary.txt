The user tested my previous fix (which was setting `allowDots: true` for `express-mongo-sanitize`). But because `express-mongo-sanitize` is version 2.2.0 (very old, from 2019), it silently ignored `allowDots: true` or evaluated the nested tree incorrectly, STILL wiping out `.responses` payloads on POST and PUT. 

In this turn, I completely removed `express-mongo-sanitize` from `server.js` and restarted the container via `quick-patch.expect`. Then I verified directly against the production container using `test-api-driver4.expect` that nested objects like `responses: { "4.1.1": {score: 3} }` now perfectly save and read back from the database.

I've fixed the issue permanently and tested it against the production MongoDB!
